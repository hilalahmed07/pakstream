PakStream Update — 2026-05-19
=============================

Drop these files over the matching paths in the target repo (preserve
directory layout; filenames are case-sensitive). Restart backend (`node`/
nodemon doesn't pick up controller/scheduler/middleware changes on its
own). Frontend rebuild via your normal `npm run build` / dev server.

Summary of changes in this drop
-------------------------------
- Single-session login enforcement (admin + regular user). Server-side
  session-token in the JWT; second login rotates it and force-logs-out
  the older browser in real time via socket and a global fetch
  interceptor on the client.
- Auth rate limiter keyed on email/username instead of IP so users on a
  shared network are not throttled together. End-Date filter in admin
  download dashboard is now inclusive of the selected day.
- Premieres run open-ended; only the admin's "End Premiere" stops them.
  Video player loops as before; backend no longer auto-ends at endTime.
- LivePremiere viewer: liveMode hides local play/pause and blocks PiP so
  viewers can't desync from the admin's broadcast.
- User home page: card click on a LIVE premiere reopens the viewer
  directly; close button is a true dismiss (no 30s reappear); admin
  ending the premiere shows a toast + auto-closes everywhere.
- Admin Premiere Management banner clears optimistically on End; End
  uses REST so authorization no longer depends on socket.userRole.
- LivePremiereControlPage: transport row stripped to only the End
  Premiere button; chat sidebar viewer count hidden for admin.
- Patch UI: eye/views removed from user card and admin table; like
  heart now matches DocumentGrid/PresentationGrid SVG style.
- Download Management filter row: User ID and Asset ID inputs removed;
  columns renamed (Name → Username, Type → Download Type).
